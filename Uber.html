Uber Autonomous Transportation Simulation System

This project applies the various design patterns discussed during the lecture namely, the factory design pattern, decorator pattern, and strategy design pattern.

The main purpose of the factory pattern is to reduce the coupling in the codebase. Specifically, in the project the use of factories allows us to easily add different entities to the simulation; inside our simulation, we have a composite factory that houses the factories for all the different entities that our simulations hold. Then at runtime when entities are added to the simulation the compiler decides which constructor to call depending on the type of entity.

The strategy design pattern's main purpose is to allow entities to choose different path-finding algorithms depending on what type of entity they are and what path-finding algorithm a user chooses. For example, drones will change the strategy that they use to navigate through the map depending on what navigation strategy the user chooses on the scheduler's website.

Lastly, we use the decorator pattern to add a battery to our drone in addition to the different movements a drone can do (jump and dance). The decorator is a wrapper that is used to add functionality to already existing base objects without having to use a complicated system of inheritance. Instead, the decorator's pattern allows us to add functionality at run-time with a wrapper around an existing object.
